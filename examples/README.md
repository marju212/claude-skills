## How to Use Examples

Each example includes:
1. **Overview**: What the example demonstrates
2. **Skills Used**: Links to relevant skill documentation
3. **Complete Code**: Fully functional implementation
4. **Tests**: Example test suites
5. **Setup Instructions**: How to run the example locally

## Contributing Examples

We welcome new examples! A good example should:
- Demonstrate 2-3 skills working together
- Include complete, working code
- Have proper error handling
- Include tests
- Be well-documented
- Represent real-world use cases

To contribute an example:
1. Create a new markdown file in this directory
2. Follow the structure of existing examples
3. Link to relevant skills from the `/skills` directory
4. Update this README with a link to your example
5. Submit a pull request
