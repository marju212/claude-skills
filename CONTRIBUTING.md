# Contributing to Claude Skills

Thank you for your interest in contributing to the Claude Skills repository! This guide will help you add new skills or improve existing ones.

## How to Contribute

### Adding a New Skill

1. **Choose the Right Category**
   - Browse the `skills/` directory
   - Select the most appropriate category for your skill
   - If no category fits, consider proposing a new one

2. **Use the Template**
   - Copy `templates/skill-template.md`
   - Rename it to match your skill (use kebab-case: `my-skill-name.md`)
   - Fill in all sections with relevant information

3. **Write Quality Documentation**
   - Provide clear, concise descriptions
   - Include working code examples
   - Add real-world use cases
   - Document common pitfalls and how to avoid them

4. **Follow the Structure**
   - Keep examples focused and practical
   - Use proper markdown formatting
   - Include code comments where helpful
   - Add links to external resources when relevant

### Updating Existing Skills

1. Identify outdated or incomplete information
2. Make specific, focused improvements
3. Test any code examples to ensure they work
4. Update the related skills section if needed

### Proposing New Categories

If you believe a new skill category would be beneficial:
1. Open an issue describing the category
2. Explain why it doesn't fit existing categories
3. Provide examples of skills that would belong to it

## Code Example Guidelines

- **Keep it simple**: Start with basic examples
- **Make it complete**: Examples should be runnable
- **Add context**: Explain what the code does
- **Use comments**: Help readers understand complex parts
- **Show output**: Include expected results when helpful

## Style Guidelines

### Markdown

- Use ATX-style headers (`#` instead of underlines)
- Include a blank line before and after headers
- Use backticks for inline code
- Use fenced code blocks with language specifiers

### Code

- Follow language-specific conventions
- Use consistent indentation (2 or 4 spaces)
- Keep line length reasonable (< 100 characters)
- Include necessary imports/dependencies

## Review Process

1. Submit a pull request with your changes
2. Provide a clear description of what you're adding/changing
3. Address any feedback from reviewers
4. Once approved, your contribution will be merged

## Questions?

If you have questions or need clarification:
- Open an issue for discussion
- Reach out to repository maintainers
- Check existing skills for examples

## Code of Conduct

- Be respectful and constructive
- Focus on improving the resource
- Help others learn and grow
- Give credit where due

Thank you for contributing to the Claude Skills repository!
